# Repository Instructions

## Release Notes

- Keep the docs release notes current for user-visible changes that are intended for `main`.
- Update [`apps/docs/release-notes/upcoming.md`](/Users/silvandiepen/Repositories/_libs/gieter/apps/docs/release-notes/upcoming.md) in the same change set when behavior, examples, documentation, supported runtimes, or developer workflow changes in a way users would care about.
- When preparing a tagged release, move the finalized `upcoming.md` content into a new versioned page under `apps/docs/release-notes/` and start a fresh `upcoming.md`.
- Write release notes for users, not commit logs. Prefer short sections such as `Added`, `Changed`, and `Fixed`, with concrete impact.
