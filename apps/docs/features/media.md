# Media and Assets

Girk copies your static files into the generated site so content pages can reference logos, images, downloads, icons, and custom files directly.

## Why You Want It

You should not need another build step just to get a logo, screenshot, stylesheet, or downloadable file into a Markdown-driven site.

## What It Covers

- `media/` and `assets/`
- logos
- favicons
- page images and thumbnails
- copied files listed in `projectCopyFiles`

## Practical Uses

- brand the site with a logo
- add screenshots to content pages
- add archive thumbnails
- ship custom CSS or JavaScript
- include downloadable files
