# 1.26.0

Released on 2026-05-04.

## Added

- Rename girk-cli to girk-runner, fix SDK types, add multi-package release.
- Add girk-sdk package with in-memory build pipeline.
- `setFetchImpl(fn)` and `FetchFn` type exported from `girk-sdk`.
- `formatDate()` — custom date formatter supporting `yy`, `yyyy`, `MM`, `M`, `dd`, `MMM` tokens.
- `path-utils` — `extname`, `basename`, `dirname`, `join`, `resolve`.
- `render.ts` — custom EJS compiler/render engine with partial support.
- `renderIcon.ts` — standalone icon rendering (extracted from pug mixin).
- `templates/registry.ts` — auto-generated in-memory template + script registry.
- `scripts/build-templates.ts` — build script to generate the template registry from `.ejs` and `.js` files.

## Changed

- **Template engine: Pug → EJS.** All templates are now EJS-based and bundled in-memory at build time. No filesystem access required at runtime. This makes girk-sdk compatible with Cloudflare Workers and other edge runtimes.
- **Configurable fetch.** `build()` now uses `globalThis.fetch` by default (Node 18+, browsers, Workers). For older Node, call `setFetchImpl(nodeFetch)` before `build()`.
- Replaced `date-fns` with a custom `formatDate()` (~60 lines, same output).
- Replaced Node `path` module with custom `path-utils` (pure string operations).
- Added `skipLibCheck: true` to tsconfig (fixes pre-existing `@types/markdown-it` type mismatch).

## Removed

- Dropped runtime dependencies: `pug`, `sass`, `purgecss`, `svgo`, `date-fns`, `node-fetch`.
- Dropped type dependencies: `@types/pug`, `@types/sass`, `@types/svgo`, `@types/node-fetch`.
- `svgo` and `purgecss` were dead dependencies (never imported in SDK code).

## Fixed

- `ci`: Add build:style step to publish workflow.
- `ci`: Update lockfile for girk-runner rename.
- `ci`: Build style assets before running tests.
