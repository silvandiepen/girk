# Examples

The repository includes sample projects that are generated by Girk itself. Use them as reference implementations rather than toy demos.

## Deployable Examples

- [example-basic.girk.dev](https://example-basic.girk.dev/) shows archives, sections, tags, redirects, copied files, custom style, and custom script.
- [example-multilang.girk.dev](https://example-multilang.girk.dev/) shows translated routes, language switching, language-specific project titles, and style overrides.

## Source Folders

- [`apps/example-basic`](https://github.com/silvandiepen/girk/tree/master/apps/example-basic)
- [`apps/example-multilang`](https://github.com/silvandiepen/girk/tree/master/apps/example-multilang)

These projects are built and verified automatically as integration fixtures, and their names are intentionally subdomain-friendly for deployments like `example-basic.girk.dev`.

## What To Look At

- how `README.md` and nested folders shape the routes
- how frontmatter sets titles, tags, icons, and archive behavior
- how assets and custom styles are copied into the output
- how translated filenames such as `guide:nl.md` create language variants
- how a project can stay entirely file-based while still generating a structured website
