# Documentation

You can run **Girk** in any folder that contains Markdown files. It derives the site directly from those files and generates the output into `public/`.

## Use

```bash
npx girky
```

## Core Model

Girk does not require a route config, a database schema, or a component tree. It derives the site from:

- the folder structure
- the Markdown filenames
- frontmatter on the pages
- optional project config in `girk.config.json` or `gieter.config.json`
