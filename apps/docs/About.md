# About

Girk is a small static site generator for Markdown-first projects. It is built around one strong idea: the files are the project model.

That means you should be able to open a folder, understand the site structure immediately, and generate a clean static site without first building a route table, database, or component system.

## Good Fits

- documentation sites
- lightweight marketing or product sites
- note collections and knowledge bases
- multilingual content trees
- archive-style content such as articles or blogs

## Project Principles

- keep content in plain files
- let the folder structure define the route structure
- make frontmatter and project config do most of the behavioral work
- generate static output that is easy to host anywhere
- keep the default output readable, minimal, and easy to override

## Practical Scope

Girk is intentionally not a full application framework. It does not try to be a CMS, a headless API layer, or a runtime app platform. It focuses on turning structured Markdown into a static website with a small set of useful behaviors around archives, tags, media, translations, and project settings.

The npm package is published as `girky`, and this documentation is generated by Girk itself from the files in this repository.
