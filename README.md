# Girk Monorepo

This repository is split into two workspaces:

- `packages/girk`: the publishable `girky` package and the `girk` CLI
- `apps/docs`: the docs site generated by `girk` itself
- `apps/example-basic`: a single-language example project with broad feature coverage
- `apps/example-multilang`: a multilingual example project with style overrules

Common commands from the repo root:

- `npm run build`
- `npm run test`
- `npm run test:examples`
- `npm run lint`
- `npm run test:run`
- `npm run serve:docs`

## npm Release

`girky` is published from GitHub Actions with `semantic-release`.

- Workflow: `.github/workflows/publish.yml`
- Branch: `master`
- Published package: `packages/girk` (`girky` on npm)
- Versioning: derived automatically from conventional commits
- Publishing mode: npm trusted publishing through GitHub Actions OIDC

Before the workflow can publish, the `girky` package must exist once on npm and npm must trust this repository/workflow pair.
Configure npm trusted publishing with:

- Organization or user: `silvandiepen`
- Repository: `girk`
- Workflow filename: `publish.yml`

Dry-run the release logic locally with:

- `npm run release:dry-run`

## Cloudflare Worker

The docs and example sites are deployed through a single Worker with static assets.

Route mapping:

- `girk.dev/*` -> docs
- `www.girk.dev/*` -> docs
- `example-basic.girk.dev/*` -> example basic
- `example-multilang.girk.dev/*` -> example multilang

Relevant commands:

- `npm run cf:build:sites`
- `npm run deploy:sites`

Deployment is handled by [deploy-sites.yml](/Users/silvandiepen/Repositories/_libs/gieter/.github/workflows/deploy-sites.yml) on pushes to `master`. It expects the `CLOUDFLARE_API_TOKEN` GitHub secret to be present.
